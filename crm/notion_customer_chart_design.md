# Notion 顧客カルテ 設計図

SORA - HAIR & EYELASH - のNotionベース顧客カルテ設計仕様書。
本ドキュメントに従ってNotion上にデータベースを構築すれば、Make.com経由のSquare自動連携と即時連携可能な構造になります。

---

## 1. 全体構成

```
📁 SORA Salon ワークスペース
├─ 👥 顧客カルテ（DB1）        ← メインDB（顧客マスタ）
├─ 📸 施術履歴（DB2）          ← 来店ごとの記録
└─ 🌿 メニューマスタ（DB3）    ← 料金・所要時間の正本
```

3つのDB間はリレーションで連動し、施術履歴に1行追加すると自動で顧客側の「最終来店日」「来店回数」「累計売上」が更新される構造です。

## 5. リレーションの構成

```
顧客カルテDB (1) ──────── (n) 施術履歴DB
                                ↕
                                (n) ──── (1) メニューマスタDB
```

- 1人の顧客 → 複数の施術履歴を持つ
- 1つのメニュー → 複数の施術履歴で使われる

これにより、顧客カルテに以下を**自動表示**できます：
- 来店回数 = 施術履歴のレコード数
- 最終来店日 = 施術履歴の最大日時
- 累計売上 = 施術履歴の料金合計
- 次回予約 = 施術履歴の最小未来日時

## 6. 構築手順（Notion作業）

実際にNotion上で構築する流れ：

### Step 1: ワークスペースとページ構造の用意

1. Notionで新規ページ作成: **「SORA Salon」**
2. その配下に空ページを3つ作成:
   - **👥 顧客カルテ**
   - **📸 施術履歴**
   - **🌿 メニューマスタ**

### Step 2: 各DBの作成

1. ページを開いて `/` → **「データベース：フルページ」** を選択
2. 上記プロパティ一覧通りにプロパティを追加
3. **重要**: メニューマスタ → 施術履歴 → 顧客カルテ の順に作成（リレーションを後から繋ぐため）

### Step 3: リレーション設定

1. **施術履歴DB** を開く
2. プロパティ追加 → **リレーション** → **顧客カルテ** を選択 → 「双方向リレーション」をONに
3. 同様に **メニューマスタ** へのリレーションも追加

### Step 4: ロールアップ設定（顧客カルテ側）

1. **顧客カルテDB** を開く
2. プロパティ追加 → **ロールアップ** → 関連: 施術履歴
3. 「来店回数」「最終来店日」「累計売上」を順に作成

### Step 5: ビュー追加

1. 各DBで **+ ビューを追加** → 上記ビュー一覧通りにフィルタ/ソートを設定

### Step 6: コネクト承認

1. 顧客カルテDBページを開く → 右上「⋯」 → **コネクト** → **SORA Salon Connector** を追加
2. 施術履歴DB・メニューマスタDBにも同様に追加

### Step 7: メニューマスタの初期データ投入

[square_services.csv](../reservation/square_services.csv) の内容をコピーして投入。

## 7. 想定運用フロー（Phase 1: 手動運用）

| 場面 | アクション |
|---|---|
| **新規お客様の来店** | カウンセリングシートを基に顧客カルテに新規ページ作成 |
| **施術完了** | 施術履歴に新規行追加（顧客選択 → メニュー選択 → 写真アップロード → メモ） |
| **施術中の確認** | スマホアプリで該当顧客カルテを開く（過去の好み・アレルギーを参照） |
| **次回来店時** | 顧客カルテで前回メモ・写真を確認 → 提案に活かす |
| **月次振り返り** | 「⚠️ 要フォロー」ビューで離反予兆顧客にLINE送付 |

## 8. Phase 2 へのバトン（API自動連携）

Phase 1で運用が安定したら、Phase 2でMake.com連携を構築。
そのために本設計では以下を**事前に組み込み済み**：

- `Square予約ID` プロパティ（重複防止）
- `作成元` プロパティ（自動 vs 手動の区別）
- `ステータス` の値設計（Squareのステータスと整合）
- メールアドレス・電話番号での顧客マッチング

これにより、Phase 1運用中に手動で作ったデータと、Phase 2で自動追加されるデータが矛盾なく統合されます。

---

## 9. 注意事項・運用ルール

1. **個人情報の取り扱い**
   - Notionの2段階認証を必ず有効化
   - スタッフ追加時は閲覧権限を最小限に
2. **写真のサイズ**
   - スマホ撮影そのままでもOK（Notionは無料プランで5MB/ファイル制限あり）
   - 多くなったらGoogle Photosとの併用も検討
3. **削除の扱い**
   - 退店された顧客は削除せず **ステータス: 離反** に変更（履歴保持のため）
4. **バックアップ**
   - 月1回 Notionの「エクスポート」機能で全DB CSV出力 → クラウド保管
